Process

How It Works

From a single sentence to a running microscopic traffic simulation in four automated steps.

Step 01
Natural Language Input
User describes a traffic scenario in plain English -- intersection type, traffic volume, signal timing, road conditions.
Step 02
Parameter Extraction
A fine-tuned LLM parses the description and extracts structured simulation parameters with high accuracy.
Step 03
Network Generation
Road networks are generated via OSM data or LLM-driven geometry, then compiled into SUMO-compatible XML files.
Step 04
SUMO Simulation
The complete simulation runs in SUMO with real-time metrics -- vehicle counts, queue lengths, travel times, and more.
System Design

Architecture

A role-separated architecture where each layer handles what it does best -- fine-tuned precision for parameter extraction, base LLM flexibility for geometry, and deterministic tooling for simulation.

Fine-tuned LLM
Parameter Extraction Layer
A fine-tuned model trained on synthetic and real traffic data to reliably convert natural language into structured simulation parameters.
  • Trained on domain-specific traffic scenarios
  • Outputs structured JSON with validated fields
  • Continuously improved via correction feedback
Base LLM
Geometry & Reasoning Layer
A general-purpose LLM handles complex spatial reasoning -- generating road network geometries, merging lanes, and interpreting ambiguous descriptions.
  • Generates node/edge geometry for custom networks
  • Handles multi-turn clarification dialogues
  • Supports Claude, GPT-4, and Gemini backends
Tool Layer
Simulation & Integration
Deterministic tooling compiles network XML, generates traffic routes, launches SUMO, and streams back real-time simulation metrics.
  • SUMO netconvert, duarouter, and TraCI
  • OSM network download and processing
  • Seoul real-time traffic API integration
Capabilities

Key Features

Built end-to-end for production readiness and continuous improvement.

Multi-LLM Support
Swap between Claude, GPT-4, and Gemini models at runtime. Compare outputs and benchmark accuracy across providers.
Fine-tuning Pipeline
Automated pipeline combining synthetic generation with real Seoul metropolitan traffic data to build high-quality training sets.
Human-in-the-Loop
Users can correct LLM outputs directly. Every correction is logged, analyzed, and fed back into the fine-tuning pipeline.
Real-time Seoul Traffic API
Integrates live traffic volume data from Seoul metropolitan sensors to ground simulations in real-world conditions.
Docker + CI/CD
Containerized deployment with GitHub Actions CI/CD pipeline. Automated testing, building, and deployment to GCP Cloud Run.
Admin Dashboard
Built-in admin panel with LLM evaluation metrics, correction rate tracking, field-level error analysis, and exportable reports.

Data Strategy

Dataset Pipeline

Three complementary data paths feed the fine-tuning pipeline, ensuring both coverage and accuracy.

01
Synthetic Generation
LLM-generated prompt-response pairs covering diverse intersection types, traffic volumes, signal configurations, and edge cases. Provides broad coverage for initial model training.
02
Real Seoul Traffic Data
Live sensor data from Seoul metropolitan traffic infrastructure converted into natural language scenarios. Grounds the model in real-world traffic patterns and volumes.
03
Correction-derived Data
Human corrections from production usage are transformed into training pairs. This closed-loop feedback continuously targets the model's actual weaknesses.
